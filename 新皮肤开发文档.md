# 新皮肤开发文档

本文档说明 Rhythm 当前皮肤系统的目录规范、元数据要求、模板回退规则，以及新增皮肤时的最小落地步骤。

## 1. 当前皮肤系统概览

- 皮肤本质上是 `src/main/resources/skins` 下的一组模板目录。
- 当前默认经典主题目录：
  - PC：`src/main/resources/skins/classic/pc`
  - 移动端：`src/main/resources/skins/classic/mobile`
- 用户皮肤设置保存在现有用户字段：
  - `userSkin`
  - `userMobileSkin`
- 设置页通过 `/settings/skin` 保存，不需要改 SQL。

## 2. 目录结构规范

新增主题建议按以下结构创建：

```text
src/main/resources/skins/
  your-theme/
    pc/
      skin.properties
      macro-head.ftl
      index.ftl
      ...
    mobile/
      skin.properties
      macro-head.ftl
      index.ftl
      ...
```

说明：

- `your-theme/pc` 表示桌面端皮肤目录。
- `your-theme/mobile` 表示移动端皮肤目录。
- `pc` 和 `mobile` 目录分别独立扫描、独立出现在设置页。
- 如果只做 PC 或只做 mobile，也可以只提供对应设备目录。

## 3. `skin.properties` 规范

每个可选皮肤目录下都必须有 `skin.properties`。

示例：

```properties
name=森林主题
version=1.0.0
memo=偏轻盈与留白的桌面端主题
theme=forest
device=pc
previewUrl=https://example.com/skinpreview/forest-pc.png
order=20
```

字段说明：

- `name`
  - 设置页展示名称。
- `memo`
  - 设置页展示描述。
- `theme`
  - 主题标识，建议与父目录一致，例如 `forest`。
- `device`
  - 只能是 `pc` 或 `mobile`。
- `previewUrl`
  - 设置页预览图地址。
  - 不填也可以，设置页会显示“暂无预览图”。
- `order`
  - 排序值，越小越靠前。
- `version`
  - 纯展示或元数据用途，可选。

## 4. 模板回退规则

当前模板加载支持“同设备 fallback”。

例如：

- `forest/pc/article.ftl` 不存在时，会回退到 `classic/pc/article.ftl`
- `forest/mobile/common/comment.ftl` 不存在时，会回退到 `classic/mobile/common/comment.ftl`

这意味着：

- 新增皮肤时，不需要一次性复制整套模板。
- 可以只覆盖你要改的页面，其余页面会自动回退到经典主题对应设备目录。

建议：

- 新皮肤至少提供：
  - `skin.properties`
  - 首页或你要定制的关键模板
  - 自己的 `macro-head.ftl`（如果你有额外头部资源）

## 5. 用户设置与兼容性

- 设置页会扫描所有 `skins/**/skin.properties`，按设备分组展示。
- 保存时：
  - PC 主题写入 `userSkin`
  - 移动端主题写入 `userMobileSkin`
- 运行时按 UA 判断当前设备，再读取用户对应字段覆盖默认皮肤。

已兼容的历史值：

- PC：`classic`、`classic-pc` 会自动映射为 `classic/pc`
- 移动端：`mobile`、`classic-mobile` 会自动映射为 `classic/mobile`

这意味着旧用户数据上线后不会因为目录升级而直接失效。

## 6. 新增皮肤推荐步骤

1. 在 `src/main/resources/skins/<theme>/pc` 或 `mobile` 下创建目录。
2. 新建 `skin.properties`，补齐 `name/memo/theme/device/order`。
3. 给 `previewUrl` 配一张预览图，方便设置页展示。
4. 只复制你要覆盖的 FTL，其余模板依赖 fallback。
5. 如果有设备差异，PC 和 mobile 分开处理，不要混用模板。
6. 本地启动后访问 `/settings/system`，检查：
   - 设置页是否能扫描到新皮肤
   - 预览图是否正常
   - 保存后当前设备是否切换成功
   - 未覆盖的页面是否正确回退到经典主题

## 7. 上线注意事项

- 改 FTL 或 `skin.properties` 后，需要重新编译资源或重启服务端，生产环境才会生效。
- 如果你新增了前端样式或脚本：
  - 改 `.scss`，不要直接改 `.css`
  - 改非 `min.js` 源码，不要直接改 `.min.js`
  - 然后执行 `yarn run build`
- 前端改动要同步评估 PC 和 mobile，两端至少要确认没有布局回归。

## 8. 建议的最小模板清单

如果你是第一次做新皮肤，建议优先从这些页面开始：

- `index.ftl`
- `article.ftl`
- `home/settings/system.ftl`
- `macro-head.ftl`
- `common/` 下你要调整的通用片段

如果只想先验证皮肤系统是否正常，最小可行方案其实是：

- 一个 `skin.properties`
- 一个你自己的 `index.ftl`

其余页面都走 fallback。
